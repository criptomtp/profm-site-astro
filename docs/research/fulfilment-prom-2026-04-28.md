# Research: Фулфілмент для Prom.ua

**Date:** 2026-04-28
**Target page:** `/fulfilment-prom/` (UA root + RU + EN siblings)
**Researcher:** Claude (research agent)
**Goal:** identify what competitors say about "fulfillment for Prom.ua" so MTP Group can win the SERP.

---

## TL;DR

- **No competitor has a dedicated `/fulfilment-prom/` landing page in the UA SERP.** Prom is mentioned as one of many channels inside generic fulfillment pages or "top operators" lists.
- **Strongest indirect competitors:** keyCRM blog (top-N list), cpashka.biz (top-11 list), horoshop.ua (what is fulfillment + top-5), keepincrm (operators + prices), osa.net.ua (what is fulfillment).
- **Average competitor depth on Prom-specific topic:** very thin — usually 1 line ("integrates with Prom") or 1 paragraph. Most don't even mention Prom API.
- **Average word count of generic fulfillment pages:** ~1,800–2,500 words. Top-list pages: 2,500–3,500 words. Recommended for our Prom-specific page: **2,000–2,400 words** (focused, not bloated).
- **Pricing market signal:** assembly 7–28 UAH per order, storage 500–650 UAH/m³/month, packaging 5–20 UAH. MTP at 18 UAH is competitive.
- **Biggest gap (where we win):** nobody explains *how Prom.ua API actually integrates with a 3PL warehouse* — endpoints, sync intervals, order statuses, token management. We can own this.

## Prom.ua API: огляд (як насправді інтегрується склад)

Source: https://public-api.docs.prom.ua/, https://prom.ua/cloud-cgi/static/uaprom-static/docs/swagger/index.html, Garazd Odoo connector docs, Torgsoft sync docs.

**Authentication:** Bearer API token. Generated in seller cabinet → **Settings → API Token Management → Create Token** (configurable scopes).

**Endpoints / functional areas (confirmed):**
- **Orders** — list new orders, update statuses, retrieve order details, payment state, payment method
- **Products** — bulk upload/update, prices, stock quantities, mapping
- **Stocks** — quantity sync per SKU
- Order statuses supported: Accepted / Completed / Cancelled (with reason)

**Sync cadences (per Garazd Odoo docs and Prom guidance):**
- Order sync: every 30 minutes (no real-time webhook)
- Product sync: every 3 hours
- Base/category sync: once daily

**Critical limitation for 3PL/fulfillment:**
- **No webhooks for new orders** — must poll every 30 min. This means our promised SLA "ship same day" needs to account for up to 30 min latency between order placement and 3PL warehouse seeing it. (no data on plans to add webhooks).
- API does not include FBO ("Fulfillment by Prom"). Prom only supports **Fulfillment by outsourcing** — i.e. the seller (or their 3PL partner) handles all logistics. This is the **opposite** of Rozetka and Kasta. **This is THE killer angle for our page.**

**Prom delivery integrations (not API but operational):** Nova Poshta, Ukrposhta, Meest, courier. "Prom Delivery" is Prom's own delivery aggregation service (30 UAH/mo + per-order). Our fulfillment can ship via any of them.

## Прогалини (де виграємо)

1. **Nobody on UA SERP explains Prom API for a 3PL.** All competitors say "API integration available" and stop. We can own the technical depth: token setup, 30-min polling, status mapping, stock push.
2. **Nobody calls out that Prom has NO FBO.** This is THE pain point of every Prom seller (vs Rozetka). Our angle: "Prom не дає свого фулфілменту — ми = ваш FBO для Prom."
3. **Nobody connects Prom commission structure (4.2–15%) with logistics costs.** Sellers don't know that fulfillment 18 UAH/order is 1.8% of a 1,000 UAH check — much smaller than Prom's commission. Math story = unique.
4. **No pricing transparency on Prom-specific operations.** Competitors quote generic "21 UAH/order". We can show: receive from supplier → store → assemble for Prom order → label → ship via NP/Укрпошта = total cost line.
5. **No competitor addresses 2024–2025 changes:** free plan closed, subdomain commission added. New Prom seller has new economics — unaddressed by content market.
6. **No competitor explains "30-min API lag" honestly** — and how a real 3PL handles SLA inside that window. Trust signal.
7. **No category-specific Prom guidance.** Top categories on Prom are Home/Garden + Electronics + Auto-moto — bulky, returns-heavy. Generic fulfillment pages don't tailor messaging to these.
8. **No competitor handles cross-marketplace inventory.** Sellers usually run Prom + Rozetka + own site. We can offer **single inventory pool, multi-channel push** (Prom every 30 min, Rozetka real-time, Horoshop instant).

  - Прийом товару від постачальника
  - Зберігання (від 18 UAH/відправка)
  - Збір замовлення з API Prom (кожні 30 хв)
  - Пакування (брендоване/нейтральне)
  - Відправка (NP/Укрпошта/Meest/Кур'єр)
  - Обробка повернень

  - Як згенерувати API токен у кабінеті Prom (4 кроки + screenshot/diagram)
  - Які дані синхронізуються: замовлення (30 хв), товари (3 год), залишки
  - Як ми мапимо статуси Prom (Accepted → Completed → Cancelled) на наш WMS
  - Чому 30-хвилинний polling — не проблема для same-day shipping
- **H2:** Економіка для продавця Prom (math story — gap #3)
  - Приклад: середній чек 1,200 UAH → комісія Prom 8% (96 UAH) + еквайринг 1.7% + наш фулфілмент 18 UAH = X% маржі

  - Дім і сад, електроніка, авто-мото, одяг, краса (+1 case study на категорію)

- **H2:** Інтеграції з логістикою (NP, Укрпошта, Meest, Prom Delivery)
