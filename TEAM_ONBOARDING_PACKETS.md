**Template 1: Guest Post Pitch** (Most common)
```

Hi [Name],

I recently read [specific article/guide] on [Company] and loved your angle on [specific insight].

[Article link]

Would this be valuable for your readers? Happy to send a draft outline.

Best,
[Your name]
```

**Template 2: Backlink Request** (For roundup/listing sites)
```
Subject: Feature Request: [Your guide] + [Their company]

Hi [Name],

I found your roundup "[Roundup Title]" and noticed it covers [topic] really well. 

Our team published a comprehensive guide on [topic] that directly aligns with section [X] of your article. We think your readers would find it valuable.

[Guide link with UTM parameters]

Would you consider adding this to your resource list?

Best,
[Your name]
```

**Template 3: Brand Mention Follow-up** (For existing mentions without links)
```
Subject: Thanks for mentioning MTP in "[Article]"

Hi [Name],

Saw that you mentioned MTP Group in your recent article "[Article]". Thanks for the shout-out!

We recently published an in-depth guide that goes deeper into the topic you covered. Thought it might be useful to link to for readers wanting more detail.

[Guide link with UTM parameters]

Happy to provide additional insights if helpful for future articles.

Best,
[Your name]
```

**Email Best Practices:**
- Personalize each email (no mass send templates)
- Keep to 3-4 short paragraphs max
- Include only 1 link (to guide with UTM params)
- Use "Best," or "Thanks," - keep casual
- Send Monday-Thursday at 9 AM (best response rates)
- Allow 5 business days before follow-up

### What to Do If...

**"No responses after 5 days"**
- Send follow-up email Day 6 with subject: `[FOLLOW-UP] Guest Post Opportunity: [Topic]`
- Keep pitch same but add: "Just following up in case this got lost in inbox"
- Assume no response after 2nd follow-up = move on

**"They want to link but want payment"**
- Flag to manager immediately
- Some companies expect payment; evaluate case-by-case
- Note: Never pay for plain backlinks (violates Google guidelines); paid content is OK if disclosed as sponsored

**"They want to counter-offer a different topic"**
- Negotiate if topic is still relevant to MTP expertise
- Confirm topic would work for profm.ua audience
- Only accept if it benefits campaign KPIs

**"They want mutual linking (reciprocal)"**
- Generally acceptable if their site is relevant
- Confirm with manager before accepting
- Note in spreadsheet as "reciprocal link"

### Troubleshooting: Common Issues

**"Build fails after adding links"**
- Check for typos in href URLs
- Ensure relative paths are correct (`/en/guide/...`)
- Validate HTML syntax (check editor for red squiggles)

**"Links work locally but break after deploy"**
- Check if deploy uses correct base URL
- Verify paths match deployed site structure
- Test on staging before production

**"PageSpeed score dropped"**
- Likely cause: new images not optimized
- Compress images <50KB each
- Check for render-blocking resources
- Lazy load below-fold images

**"Build succeeds but Core Web Vitals still poor"**
- CWV takes 24+ hours to update
- Focus on optimization checklist (LCP, INP, CLS)
- Verify changes in Lighthouse audit
- Deploy and monitor for 1 week

---

### What NOT to Do

❌ Don't change site structure (no URL rewrites)  
❌ Don't add new content (only optimize existing)  
❌ Don't alter navigation (keep user experience same)  
❌ Don't merge multiple unrelated changes (keep scope tight)  
❌ Don't deploy without testing locally first

### Key Metrics Definitions

**Organic Sessions:** Visits to profm.ua from Google Search (not paid ads)

**Backlink Referral Sessions:** Visits that came through acquired backlinks (tracked via UTM `utm_source`)

**Engagement Rate:** % of sessions with meaningful interaction (scroll, click, form submit)

**Response Rate:** (Responses received / Emails sent) × 100%

**Link Rate:** (Backlinks placed / Responses received) × 100%

**PageSpeed Score:** Google's 0-100 performance metric (100 = best)

**LCP (Largest Contentful Paint):** Time for largest visual element to load (target <2.5s)

**INP (Interaction to Next Paint):** Time for page to respond to user interaction (target <200ms)

**CLS (Cumulative Layout Shift):** Visual stability of page (target <0.1)
